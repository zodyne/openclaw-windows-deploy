# 七种病理目录（早期信号与对策）

1. 术语稀释——项目荣誉词（如"Golden验证"）从特定数据源悄悄泛化为任意参考
   数据。信号：报告中荣誉词覆盖面变大。机器对策：TIERS 定义入锁 +
   evidence_must_reference。人工对策：抽查顶档证据链。

2. 静默顶替/降级——难走的验证路径被换成容易的替代路径，照常报绿。信号：
   某项突然从卡住变顺利、方法描述变模糊。机器对策：R5 + blocked 合法出口。
   人工对策：问"数据源和方法与上次完全一样吗"。

3. 档位合并——"Golden/合成验证 30/30 ✅"。信号：斜杠、顿号连接的档位名、
   一次性大批量绿灯。机器对策：封闭枚举 + 汇报只能脚本生成。

4. 无据声明——✅ 没有对应产物。机器对策：R3 + 校验器的 evidence 存在性与
   verify_cmd 检查。人工对策：定期篡改测试，确认拦截仍有效。

5. 范围重构——未完成的验收项被论证为"目标之外/额外/可选"。信号：这些修饰词
   出现在原 DoD 条目上。机器对策：DoD 在锁定宪章内 + 改判走 CR。

6. 记忆固化——虚假结论写进 MEMORY.md，后续会话当既定事实引用，漂移被洗白。
   机器对策：R7 指针制。人工对策：retrofit 时记忆消毒并留 diff。

7. 泛问话术——"进展如何"得到叙事化好消息。人工对策：只问文件级可证伪问题
   （"X.bin 被哪条 verify_cmd 加载过，evidence 在哪"）；汇报只接受
   gen_report 原文。
